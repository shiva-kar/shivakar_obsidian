### **Part 1: 25 Short-Answer Questions (Definitions & Concepts)**

**1. What is a "Cognitive Model" in HCI?** A cognitive model is a readily manipulable depiction of a user’s cognitive capacities. These models focus on behavioral representations rather than the physical environment in which the user operates. They are significant because they offer a means of comprehending individuals’ cognitive capacities to guide system evaluation and design. This allows designers to predict how users will interact with an interface before it is built.

**2. Define the acronym "GOMS."** GOMS stands for Goals, Operators, Methods, and Selection Rules. It makes use of an explicit model of cognitive processes developed through research to describe what users do and how they accomplish it. GOMS is a family of modeling approaches that predicts how long a task will take to complete, given specific task descriptions and user interfaces.

**3. What is the "Model Human Processor" (MHP)?** The Model Human Processor is a simplified view of the psychological processes involved in human cognition, based on empirical evidence from psychological studies. It is designed to be usable by non-psychologists, such as designers and evaluators. The MHP consists of a set of processors, memories, and principles of operation that interact to simulate human behavior.

**4. Define the "Perceptual System" within the MHP.** The perceptual system processes data from the eyes and hearing into working memory. It stores information in visual and auditory image stores as symbolic representations. For instance, it might code a car horn as a "loud warning". This system focuses on the actual physical intake of stimuli from the world.

**5. Define the "Cognitive System" within the MHP.** The cognitive system decides how to react by combining information from working memory and long-term memory. It analyzes sensory information, recalls relevant procedures or rules, and chooses an action. For example, recognizing a warning sound and deciding to run to the pavement based on evasive action rules is a cognitive system function.

**6. Define the "Motor System" within the MHP.** The motor system carries out the physical reactions to choices made by the cognitive system. It transforms mental decisions into muscular movements. An example of its function is the actual act of sprinting toward a sidewalk after the cognitive system has determined that it is the necessary course of action.

**7. What is a "Goal" in the GOMS model?** A goal is anything the user wishes to achieve, such as creating a directory or deleting a file. Goals are structured hierarchically, meaning a main objective is made up of several smaller subgoals. GOMS assumes that users are aware of their goals and evaluates their activities in relation to those specific objectives.

**8. Define "Operators" in the context of GOMS.** Operators are the fundamental perceptual, motor, or cognitive activities required to alter a user’s mental state or environment. They represent the lowest level of a GOMS analysis and cannot be broken down into smaller actions. Operators must occur sequentially, as GOMS models behavior as a series of non-simultaneous actions.

**9. What are "Methods" in a GOMS analysis?** Methods are sets of actions or operators that are grouped together to achieve a specific objective. Like goals, methods can contain subgoals. A fundamental tenet of GOMS is that methods are routine and learned through practice, meaning the user does not need to perform active problem solving to execute them.

**10. Define "Selection Rules" in GOMS.** Selection rules specify which method to use when there are multiple ways to achieve a single goal. They are typically formatted as IF... THEN statements, such as "IF the restaurant accepts credit cards, THEN pay by credit card". GOMS assumes error-free performance, meaning the user always applies the correct selection rule.

**11. What is the "Keystroke Level Model" (KLM)?** The Keystroke Level Model is the lowest level of GOMS analysis. It provides quantitative predictions of the time an expert operator needs to complete a task, assuming error-free performance. Task execution is described through specific operators like keying, pointing, and homing, each assigned a specific duration based on human performance data.

**12. Define the "Homing (Th)" operator in KLM.** The homing operator represents the time required for a user to switch their hands between different input devices, such as moving from the keyboard to the mouse or vice versa. In KLM task calculations, this physical-motor operator must be included whenever the task requires the user to change their physical interaction point.

**13. What is "Interacting Cognitive Subsystems" (ICS)?** ICS is a complex framework assuming human perception, thought, and behavior are composed of separate, connected information processing units. It provides a richer description of human cognition than GOMS by modeling the interaction of multiple subsystems. Each subsystem functions independently within its own processing area, transforming specific types of information.

**14. What are the three components of an ICS processing area?** Each ICS subsystem consists of three primary components: sensory (visual and auditory stimuli), representational (information codes), and effector (body movement). Subsystems also contain a memory record and a set of transformations. This structure allows the subsystem to receive input, store it, and transform it into different outputs or codes.

**15. Define "Computer-Supported Cooperative Work" (CSCW).** CSCW is a field that examines the effects of computers on cooperative working in general and groupware systems in particular. It recognizes that multiple people are involved in systems like email or conference calls. All systems, to some degree, impact and are influenced by the social groupings and circumstances where they are situated.

**16. What is "Task Analysis"?** Task analysis is the process of examining how people carry out their work, including their actions, the objects they act upon, and the knowledge they require. It focuses on defining user performance rather than computer performance. It involves observing current systems and procedures to understand the hierarchy of tasks needed to reach a useful goal.

**17. Define "Task Decomposition."** Task decomposition is a task analysis methodology that examines how a large task is divided into smaller subtasks. It also investigates the order in which these subtasks are completed. A primary example is Hierarchical Task Analysis (HTA), which produces a hierarchy of tasks and the plans required to execute them.

**18. What is the "Lexical Level" of computer dialogue?** The lexical level is the lowest level of computer language, dealing with the actual physical inputs and outputs. It involves the form of the icons on the screen and the specific keys hit by the user. It is comparable to the spelling and sound of individual words in human language.

**19. Define the "Syntactic Level" of computer dialogue.** The syntactic level refers to the arrangement and hierarchy of inputs and outputs. It is the level most frequently associated with the term "dialogue" in user interface design. This level is comparable to the sentence formation and grammar rules found in human language.

**20. Define the "Semantic Level" of computer dialogue.** The semantic level refers to the meaning of the dialogue and its effect on the computer’s internal data structures or the external environment. It captures the significance that each participant assigns to the communication. It is comparable to the actual meaning conveyed by a sentence in human conversation.

**21. What is the "PIE Model"?** The PIE model is a formal black-box interaction model that discusses user inputs and outputs without depicting internal system architecture. It distinguishes between a system’s transient appearance (display) and its lasting outcome (result). The acronym represents the relationship between the Program (input history), the Interpretation function, and the resulting Effect.

**22. Define "Observability" in interaction models.** Observability is a principle describing the user's ability to infer the system's state or the final outcome from the current display. In the PIE model, it asks whether the user can determine "what they see is what they get". It involves the relationship between the transient display and the lasting result.

**23. What is the "Gone Away for a Cup of Tea Problem"?** This is a predictability problem where a user returns to a system after a break and must be able to determine the current state from the display. If the display does not indicate the current state or the result of future actions (such as which object is on top), the user must rely on memory. A predictable system allows the user to determine the state just by looking.

**24. Define "Hypertext."** Hypertext is a non-linear text structure that organizes information as a mesh of pages and links instead of a single line. It allows users to follow their own path through the content rather than a direction dictated by the author. Links within the content, known as hot-spots, facilitate this navigation.

**25. What is "Ubiquitous Computing"?** Ubiquitous computing is a paradigm where information processing is integrated into everyday tasks or objects. It involves microprocessors embedded in electronic devices to communicate information constantly. Unlike desktop computing, it allows computing in any format, at any time, and from any location.

### **Part 2: 10 Long-Answer Questions (Theories, Models, & Processes)**

**1. Explain the components and predictive capabilities of the GOMS model.** GOMS provides a structured way to analyze how users interact with a system to achieve specific goals:

- **Goals:** What the user wants to accomplish (e.g., "save file"). These are structured hierarchically into subgoals.
- **Operators:** The most basic actions (motor, cognitive, or perceptual) that cannot be further decomposed.
- **Methods:** Learned, routine procedures consisting of a sequence of operators to achieve a goal.
- **Selection Rules:** Conditional (IF... THEN) statements that determine which method to use when several are available.
- **Predictive Power:** Using the **Model Human Processor**, GOMS can predict task completion times with 10% to 20% accuracy for expert users.
- **UI Evaluation:** It identifies **consistency** (how similar tasks are performed) and **coverage** (whether the UI supports all necessary tasks).

**2. Describe the three interacting subsystems of the Model Human Processor (MHP).** The MHP provides a simplified cognitive architecture to predict user behavior through three subsystems:

- **Perceptual System:** This handles sensory input from the eyes and ears, storing it in visual and auditory image stores as symbolic codes.
- **Cognitive System:** The "central processor" that combines new sensory data with information from **working memory** and **long-term memory** to make decisions.
- **Motor System:** This executes the physical response determined by the cognitive system, such as a hand movement or a keypress.
- **Parameters:** Each system is characterized by **processing cycle time** (how long it takes to handle one piece of data), **store capacity**, and **decay time** (how long memory lasts).
- **Textual Diagram Flow:** Sensory Input -> Perceptual Processor -> Working/Long-term Memory -> Cognitive Processor -> Motor Processor -> Physical Action.

**3. Detail the Architecture of Interacting Cognitive Subsystems (ICS).** ICS models the human mind as a network of specialized processing units:

- **Subsystem Independence:** Each unit functions independently in its own domain, such as visual, acoustic, or limb control.
- **Information Transformation:** Subsystems contain **transformations** that change input into different codes (e.g., changing visual input into an object representation).
- **Memory Records:** Every subsystem stores a copy of incoming data in a **memory record** for future retrieval.
- **Data Network:** Subsystems are connected via a network where data in one format (like **MPL** - speech-based code) is moved to another subsystem to be converted into meaning (**PROP** - propositional meaning).
- **Application:** Designers use ICS to map the "mental resources" required for a task by counting the number of transformations needed.

**4. Discuss the characteristics of Face-to-Face Communication and their transfer to HCI.** Face-to-face interaction is a complex multimodal process that users expect systems to mirror:

- **Personal Space:** Social conventions regarding physical distance (proximics) transfer to mediated communication; users feel uncomfortable if these norms are breached.
- **Eye Contact and Gaze:** Gaze indicates interest and boredom; maintaining **social presence** often requires simulating direct eye contact.
- **Gestures and Body Language:** We use physical movements to point to objects ("this" and "there"), a concept called **deictic aid**.
- **Back Channels:** Non-verbal cues (nods, "mhm") provide feedback without taking a turn in the conversation.
- **Impact of Technology:** Large monitors or keyboards can obstruct these cues, leading researchers to design shared workspaces where equipment is integrated into desks to facilitate natural eye contact.

**5. Explain Speech Act Theory and its application in "Conversation for Action."** This theory focuses on the functional intent of utterances rather than just their propositional meaning:

- **Illocutionary Point:** The intended effect of a statement (e.g., a request, a promise, or an assertion).
- **Adjacency Pairs:** Conversations are often structured as pairs of related turns (e.g., Question-Answer or Offer-Acceptance).
- **Grounding:** The process by which participants establish a **common ground** or shared understanding of the context.
- **State Diagrams:** Systems like "The Coordinator" use state diagrams to track the status of a "Conversation for Action," showing how a request leads to a promise, which leads to fulfillment.
- **Textual Diagram Flow:** Request -> Promise -> Fulfillment -> Acknowledgment.

**6. Outline the process and utility of Hierarchical Task Analysis (HTA).** HTA is a foundational method for decomposing complex work into manageable parts:

- **Hierarchy Generation:** A high-level task is broken into subtasks, which are further broken down until a "stopping criterion" is reached.
- **Plans:** These specify the conditions and sequence for subtasks. For example, "do 1, then 2; when bag is full do 3".
- **Refinement:** Analysts check for imbalances or omissions, such as realizing a "warm pot" step is missing from a tea-making hierarchy.
- **P x C Rule:** Expansion stops when the **probability of error (P)** multiplied by the **cost of error (C)** is below a threshold, meaning no further training or design detail is needed for that subtask.
- **Textual Diagram Flow:** Main Goal -> [Subtask 1 + Plan] -> [Subtask 2 + Plan] -> [Subtask 3].

**7. Compare the syntactic and semantic levels of Human-Computer Dialogue.** Dialogue design requires a distinction between the structure of the exchange and its result:

- **Syntactic Level:** Focuses on the sequence and rules of the conversation—the "grammar" of the interface.
- **Semantic Level:** Focuses on the "meaning" or the functional change the dialogue causes in the computer's data or the world.
- **Separation of Concerns:** Using specific **dialogue notations** (like STNs or Production Rules) allows the designer to analyze the dialogue syntax without getting bogged down in implementation code.
- **Linkages:** To create a prototype, the syntactic dialogue must be linked to **semantics** through notation-specific registers or attachments to programming languages like C.

**8. Describe the PIE Model and its principles of transparency.** The PIE model is used to formally analyze the "observability" of a system:

- **Black-Box Components:** It maps the **P**rogram (command history) through an **I**nterpretation function to an **E**ffect.
- **Display vs. Result:** The **Display** is the transient screen appearance, while the **Result** is the permanent outcome (like a printout).
- **Transparency (Result):** A system is transparent if the user can fully determine the final result just by looking at the current display.
- **Transparency (State):** A system is transparent if the current internal state is fully visible on the display, eliminating hidden modes.
- **Textual Diagram Flow:** Input History (P) -> Interpretation (I) -> Effect (E) -> [Display (D) + Result (R)].

**9. Explain the "Smart DEI" Holistic Framework for Ubiquitous Computing.** This framework provides a foundation for studying and creating pervasive systems:

- **Smart Devices:** Multi-functional, mobile portals (like phones) that provide access to services.
- **Smart Environments:** The physical infrastructure and sensors that allow systems to become "context-aware".
- **Smart Interaction:** The use of intelligent agents and self-organizing models to facilitate natural human-system engagement.
- **Interaction Model:** Focuses on three directions: system-to-system (C2C), system-to-human (HCI), and system-to-physical world (CPI).
- **Service Life Cycle:** Involves the continuous stages of **Creation**, **Execution**, **Maintenance**, and **Dissolution** of distributed services.

**10. Discuss Ethical Issues in HCI as presented in the textbook.** As technology integrates into life, designers must address several ethical imperatives:

- **Privacy and Informed Consent:** Users must know what data is collected (surveillance) and provide express permission for its use.
- **Fairness and Bias:** Designers must identify and eliminate **algorithmic bias** in training data to ensure equitable outcomes for all user groups.
- **Dark Patterns:** Ethical design avoids using cues that manipulate users into making decisions against their best interests.
- **User Well-Being:** Systems should be designed to minimize **addiction and overuse**, prioritizing mental health over engagement metrics.
- **Sustainability:** Ethical HCI includes energy-efficient interface design and responsible end-of-life recycling for electronic devices.

### **Part 3: 6 Applied/Design Tasks (Scenarios & Analysis)**

**1. Performing a KLM Time Prediction**

- **Scenario:** An expert user needs to insert the word "brown" into the sentence "The quick fox jumps..." after the word "quick." The cursor is currently at the end of the sentence.
- **Steps:**
    1. **Homing (Th):** Move hand from keyboard to mouse.
    2. **Pointing (Tp):** Move mouse to the insertion point after "quick".
    3. **Button Click (Tb):** Click the mouse button to set the cursor.
    4. **Homing (Th):** Move hand back to the keyboard.
    5. **Mental Preparation (Tm):** Formulate the word "brown" to be inserted.
    6. **Keying (Tk):** Type the five characters of "brown" (5 * Tk).
    7. **Total Time Calculation:** Sum the standard times for Th + Tp + Tb + Th + Tm + (5 * Tk).

**2. Developing a Hierarchical Task Analysis (HTA)**

- **Scenario:** Analyze the task of "Preparing a Vacuum Cleaner for Use."
- **Steps:**
    1. **Identify Main Goal:** Task 0: Prepare Vacuum Cleaner.
    2. **Decompose into Subtasks:** 1. Get vacuum out of closet; 2. Attach tools; 3. Plug in cord.
    3. **Develop a Plan:** Plan 0: Do 1, then 2. If cord is short, do 3.
    4. **Further Decomposition:** Break Subtask 2 into 2.1 Identify tool; 2.2 Snap onto hose.
    5. **Apply Stopping Criterion:** Stop expanding when tasks are basic physical actions that require no further explanation.

**3. Applying the PIE Model to Predictability**

- **Scenario:** You are designing a layered graphics program. Two shapes (a circle and a square) are overlapping.
- **Steps:**
    1. **Identify the "Gone Away for a Cup of Tea" Problem:** A user returns and sees the overlap but cannot tell which shape is "on top".
    2. **Check Observability:** If the user clicks, they don't know which shape will be selected.
    3. **Design an Observable Effect (O):** Add a visual highlight or a "Layers Palette" that shows the stacking order.
    4. **Verify Transparency:** Ensure that any future command (result) can be predicted solely by looking at this new display element.

**4. Analyzing an ICS Transformation Sequence**

- **Scenario:** Describe the cognitive path when a user sees a "Save" icon and decides to click it.
- **Steps:**
    1. **Sensory Input:** The user views the icon via the **Visual Subsystem (VIS)**.
    2. **Object Representation:** VIS transforms the input into an **Object (OBJ)** representation (e.g., "square icon with floppy disk image").
    3. **Meaning Extraction:** The OBJ representation is sent to the **Propositional Subsystem (PROP)** to extract meaning ("this is the save button").
    4. **Action Planning:** The user decides to save, sending the instruction to the **Limb Subsystem (LIM)**.
    5. **Effector Output:** LIM transforms the intent into **Motor (MOT)** hand movements to click the mouse.

**5. Building a Knowledge-Based Taxonomy**

- **Scenario:** Categorize the objects and actions involved in "Withdrawing Cash from an ATM."
- **Steps:**
    1. **Identify Physical Objects (XOR):** ATM card, Keypad, Cash dispenser, Screen.
    2. **Identify Data Objects (AND):** PIN number, Account balance, Withdrawal amount.
    3. **Identify Actions (XOR):** Insert card, Enter PIN, Select amount, Take cash, Take receipt.
    4. **Define Hierarchy:** Group "Insert card" and "Enter PIN" under a high-level "Authentication" node.

**6. Evaluating a Speech Interface Design**

- **Scenario:** A designer wants to replace a mouse-driven drawing tool with a voice-activated one.
- **Steps:**
    1. **Analyze Cognitive Load:** Recognize that speech requires "acoustic memory," which can interfere with planning and problem-solving compared to hand-eye coordination.
    2. **Identify Obstacles:** Note the potential for background noise interference and the slow pace of voice output.
    3. **Evaluate Advantages:** Determine if the "hands-busy" or "eyes-busy" condition applies to this drawing task.
    4. **Propose a Hybrid Solution:** Use voice for choosing tools (which avoids moving the cursor to a palette) and the mouse for the actual drawing (direct manipulation).
