# PERSONAL ROADMAP (BCA → MCA → EARLY CAREER)

**Purpose:**  
Become a **reliable systems/backend engineer** with C, C++, Java, Python.  
Not flashy. Not trendy. **Employable long-term.**

---

## GLOBAL RULES (DO NOT BREAK)

1. **College syllabus ≠ learning**
    
    - Syllabus = minimum compliance
        
    - Real learning = outside class
        
2. **One primary language at a time**
    
    - Others stay in maintenance mode
        
3. **No tutorial hopping**
    
    - One resource per topic
        
    - Finish it or drop it permanently
        
4. **No “I’ll understand later”**
    
    - If you don’t understand now, you are lying to yourself
        
5. **Projects > notes**
    
    - If it doesn’t compile/run, it doesn’t exist
        
6. **Debug yourself first**
    
    - Google only after you isolate the problem
        
7. **No new languages**
    
    - C, C++, Java, Python only
        
    - Everything else is noise
        

---

# DAILY / WEEKLY BASELINE (NON-NEGOTIABLE)

### Weekdays

- 30 min reading (concepts)
    
- 90 min coding
    
- 30 min debugging / refactoring
    

### Weekends

- 1 long session (4–6 hrs)
    
- Only ONE task
    
- Finish something measurable
    

If you skip → you compensate.  
No “rest day” logic.

# BCA ROADMAP (3 YEARS)

## YEAR 1 — C FOUNDATION (SEM 1–2)

### PRIMARY: C

### SECONDARY: none (Python max 20%)

### YOU MUST BE ABLE TO:

- Trace pointers on paper
    
- Predict segmentation faults
    
- Explain stack vs heap clearly
    
- Write DS from scratch
    
- Split code into `.h` / `.c`
    
- Use `gdb`, `valgrind`
    
- Explain compilation stages
    

### DO THIS

- Reimplement:
    
    - Array
        
    - Linked list
        
    - Stack
        
    - Queue
        
- Write:
    
    - File-based record system
        
    - Word count tool
        
- Break your code intentionally
    
- Fix memory leaks
    

### DO NOT

- Touch frameworks
    
- Chase “apps”
    
- Compare yourself to YouTubers
    
- Skip pointer problems
    

If C is weak → everything after is garbage.

---

## YEAR 2 — OOP TRANSITION (SEM 3–4)

### PRIMARY: Java

### SECONDARY: Python

### MAINTAIN: C

---

### JAVA — NON-NEGOTIABLE OUTCOMES

You must understand:

- Classes vs interfaces
    
- Encapsulation (for real, not definitions)
    
- Object lifecycles
    
- Exception handling
    
- Collections
    
- Clean structure
    

You must be able to:

- Design a small system on paper
    
- Implement it cleanly
    
- Read someone else’s Java code without panic
    

---

### PYTHON — LIMITED SCOPE

Python is **not your identity**.

You use it for:

- Scripts
    
- File processing
    
- Simple APIs
    
- Data handling
    

No ML obsession yet.

---

### UNIX / SHELL

You must:

- Navigate filesystem blindly
    
- Understand permissions
    
- Use pipes and redirects
    
- Write basic shell scripts
    

This is not optional.

---

### DO NOT

- Write messy Java “just to pass”
    
- Ignore structure
    
- Overlearn Python syntax tricks
    
- Skip Unix labs
    

---

## YEAR 3 — INTEGRATION (SEM 5–6)

### PRIMARY: Java

### SECONDARY: Python

### MAINTAIN: C++

---

### MAJOR PROJECT (CRITICAL)

Your major project must:

- Have a backend
    
- Use a real DB
    
- Handle errors properly
    
- Be version-controlled
    
- Be deployable (even locally)
    
- Have documentation
    

If your project is:

- Only UI → bad
    
- Copy-paste → useless
    
- Toy → waste of time
    

This project becomes your **baseline confidence**.

---

### BY END OF BCA YOU MUST:

- Be intermediate in all 4 languages
    
- Be comfortable switching languages
    
- Understand systems at a basic level
    
- Not be confused by syntax
    

Not expert. **Competent.**

# MCA ROADMAP (2 YEARS)

## MCA YEAR 1 — DEPTH (SEM 1–2)

### PRIMARY: C++

### SECONDARY: Java

### PYTHON: tool only

---

### C++ — THIS IS WHERE YOU LEVEL UP

You must understand:

- RAII
    
- Ownership
    
- STL (not memorized, understood)
    
- References vs pointers
    
- Basic concurrency
    
- Performance implications
    

You must be able to:

- Implement DS in C++
    
- Write medium-sized programs
    
- Read large codebases
    
- Debug runtime issues
    

If you want Unreal Engine:

- Only AFTER fundamentals
    
- As side work only
    
- No game obsession
    

---

## MCA YEAR 2 — SPECIALIZATION (SEM 3–6)

### FOCUS AREAS

Pick **ONE primary direction**:

- Backend & distributed systems
    
- Systems / performance
    
- Infra / networking
    

### ELECTIVES — PICK THESE

- Design Patterns
    
- Software Architecture
    
- Distributed OS
    
- Computer Networks
    
- Cloud Computing
    
- Network Programming
    

ML:

- At most 1–2 subjects
    
- No ML ego
    

---

### DISSERTATION (VERY IMPORTANT)

Your dissertation must:

- Be backend or systems-heavy
    
- Use Java or C++
    
- Involve scale, concurrency, performance, or networking
    
- Be something you can defend technically
    

No fluffy topics.
